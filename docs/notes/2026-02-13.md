## 1️⃣ Steger Ridge Detection Tuning

### Key Parameter Adjustments

- Increased `sigma` from 1.2 → 2.0
  - Significantly reduced noise in Hessian derivatives.
  - Improved stability of subpixel offset `t`.

- Increased `t_max` from 0.5 → 1.0
  - Critical fix for broken centerlines.
  - Prevented valid ridge points from being rejected.
  - Restored continuity.

- Maintained moderate `ridge_thresh` (~6–10).

### Insight

The discontinuities were **not** caused by NMS or linking.
They were caused by overly strict subpixel refinement rejection:

```
ridge_mask &= (abs(t) <= t_max)
```

Allowing `|t| ≤ 1.0` aligns with Steger’s derivation and fixed the issue.

## 2️⃣ Quantitative Subpixel Validation

Ran evaluation over 200 synthetic frames.

Results:

- N errors: 83,800
- Mean error: 0.319 px
- Median error: 0.250 px
- P90: 0.468 px
- P95: 0.516 px
- Max: 37.98 px (outliers due to missed detections)

### Interpretation

- Typical localization accuracy ≈ 0.25–0.35 px.
- 95% of detections within 0.52 px.
- Confirms true subpixel performance.
- Outliers correspond to complete local detection failure.

Conclusion: ridge detection is stable and production-ready.

## 3️⃣ Centerline Extraction (Orientation Agnostic)

Implemented PCA-based centerline reconstruction:

1. Convert ridge points (y,x) → (x,y).
2. Perform PCA to obtain dominant stripe direction.
3. Project points into (s,t) local frame:
   - s = along stripe
   - t = across stripe
4. Bin along s.
5. Select strongest ridge point per bin.
6. Interpolate small gaps.
7. Smooth t(s).
8. Map back to image coordinates.

Advantages:
- No per-column assumptions.
- Works for arbitrary stripe angle.
- Robust to curvature.
- Handles sparse detections.

This produces an ordered, continuous centerline suitable for triangulation.
